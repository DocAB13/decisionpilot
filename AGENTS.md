## Working Rules

- Follow the Handbook exactly.
- Follow IR01 in order.
- Never redesign architecture.
- Reuse existing code whenever possible.
- Implement one task at a time.
- Make the smallest possible change.
- Run verification after every task.
- Commit after every completed task.
- Stop and wait for approval before continuing.

## Response Format

- **Verification** — what was run and the result
- **Files changed** — list of created or modified files
- **Commit** — hash and message
- **Waiting for approval**
